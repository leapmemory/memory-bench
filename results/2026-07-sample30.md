# Run: free-tier quick sample (July 2026)

The smallest corpus in this repo: 28 turns, 8 probes, sized so a full run
costs about $1.40 in LeapMemory credits. Reproduce with:

    python3 bench.py --corpus corpus/sample_30.jsonl --probes probes/sample_30.yaml --ingest

## Result

| Metric | Value |
|---|---|
| Probes | 8 (incl. one abstention probe) |
| Accuracy, pasted history | 8/8 |
| Accuracy, LeapMemory recall | 8/8 |
| Scored (both correct) | 8 |
| Median input-token reduction | 78.2% |
| Range | 58.3% - 90.3% |
| Model, both conditions | claude-sonnet-4.6, temperature 0 |
| Token source | provider-billed prompt_tokens |

## Why this number is the lowest in the repo

Reduction is a ratio, and this haystack is tiny: the full paste is only about
725 tokens, while a recall context has a floor cost of 70-300 tokens
regardless of corpus size. The smaller the history, the larger the fraction
recall represents. Across the three corpora the pattern is the point:
28 turns -> 78%, 98 turns -> 92%, and a dense 25k-token conversation -> 83%,
with the paste condition growing linearly with history while recall stays
flat. The larger your history, the larger the saving; at some history size
the paste stops fitting in a context window at all.
